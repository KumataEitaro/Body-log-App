# BodyLog テストシナリオ

## 実行方法
- 単体テスト: `npm test`（Vitest、32件）
- 結合テスト(自動・認証不要): `node tests/integration.mjs`（13件、本番に対して実行）
- 結合テスト(手動・要ログイン): 下記シナリオ表をブラウザで実施

## 自動テストのカバー範囲
| 区分 | 内容 |
|---|---|
| 単体 calc | BMR計算(男女/丸め)・目安kcal(運動5段階/補正)・判定5段階の全境界値・CSSクラス対応・JST日付 |
| 単体 API | 未認証401・キー未設定500・空入力400・回数上限429・正常系(+カウント増)・モデルフォールバック・全滅時502・不正画像の除外・不正JSON400 |
| 結合 A | 未ログイン時の全ページリダイレクト、ログイン画面表示 |
| 結合 B | AI APIの認証ガード |
| 結合 C | RLS: 匿名からentries/profilesが読めない・書けない |
| 結合 D | migration-2の反映(photo_urls列・ai_usageテーブル) |
| 結合 E | 写真ストレージへの匿名アクセス防止 |

## 手動シナリオ（要ログイン・スマホ推奨）

| # | シナリオ | 手順 | 期待結果 |
|---|---|---|---|
| M-1 | 新規登録→初期設定 | 新しいメールでアカウント作成 | 確認メール無しで即オンボーディング画面→保存で入力画面へ |
| M-2 | AI解析(テキスト) | 「朝は納豆ごはん、昼はラーメン」→AIで解析 | 品目表+合計が表示、残り回数が1減る |
| M-3 | AI解析(写真) | 食事の写真を添付して解析 | 写真の内容が品目に反映される |
| M-4 | 体重・運動の抽出 | 「体重75.2kg、ジムで筋トレした」を含めて解析 | 表示欄に体重75.2/運動量が自動で入る |
| M-5 | 保存 | 解析結果を保存 | 「保存しました。目安◯/差◯/判定◯」が出る |
| M-6 | 上書き | 同じ日付でもう一度保存 | エラーなく上書きされる |
| M-7 | ダッシュボード | 保存後にダッシュボードを開く | KPI・テーブル・チャートに当日分が反映 |
| M-8 | 手直し | 解析後「手直しする」で数値変更→反映→保存 | 変更後の値で保存される |
| M-9 | 回数制限 | (確認のみ) ボタンの「今日あと◯回」表示 | 解析のたびに1ずつ減り、0で押せなくなる |
| M-10 | データ分離 | 友達のアカウントでダッシュボードを開く | 自分の記録だけが見える(他人の分が見えない) |
| M-11 | ログアウト/再ログイン | ログアウト→ログイン | データがそのまま残っている |
| M-12 | 過去データ取込 | 設定→JSONを貼って取込 | 「◯件を取り込みました」→ダッシュボード反映 |
| M-13 | 品目の分量修正 | 解析後、品目表の分量を「50g」→「110g」に変更 | その品目のkcal/PFCが2.2倍にスケール、合計も自動再計算 |
| M-14 | 品目の数値修正 | 品目のkcalを直接変更 | 合計kcalが即座に追従 |
| M-15 | 品目の削除/追加 | ×で品目削除、「＋品目を追加」で行追加 | 合計が自動再計算される |
| M-16 | マイ食品登録 | 設定→マイ食品・レシピで「パルテノ 1個 90kcal P10 F4 C5」を登録 | 一覧に表示され、入力画面にチップが出る |
| M-17 | チップから追加 | 入力画面で「＋パルテノ」チップをタップ | AI不要で品目に追加、合計に反映 |
| M-18 | レシピの分量質問 | 「野菜鍋」をレシピ登録→チャットに「野菜鍋」とだけ書いて解析 | 「どのくらい食べましたか？」の質問が表示される。「野菜鍋 1/3」なら登録値の1/3で計算される |
| M-19 | 複数回記録 | 朝食を保存→昼食を保存 | 保存のたびに入力欄がクリアされ、フィードに時刻つきで2件並ぶ。上部の「摂取済み/残り」が合算される |
| M-20 | 運動の加算 | 「筋トレ1時間」を保存→夜「ランニング」を保存 | 目安が +150 → +550 と加算されていく |
| M-21 | フィード1件削除 | フィードの×で1件削除 | 合計・残りが即座に再計算される |
| M-22 | 基礎代謝の内訳表示 | 入力画面を開く | 上部に「基礎代謝×係数＋運動＝目安」「摂取済み/残り」が表示される |
| M-23 | 旧形式の日への追記 | 7/13以前の日付に新しい記録を保存 | 旧まとめ分が「（旧形式から移行）」としてフィードに現れ、合計が壊れない |
| M-24 | 記録の編集 | フィードの✎をタップ→数値を修正→「編集を保存する（上書き）」 | その記録が上書きされ、上部の合計・ダッシュボードも即再計算。キャンセルで破棄できる |
