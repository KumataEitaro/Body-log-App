# BodyLog 機能一覧（UI リニューアル用インプット）

これは減量トラッカー「BodyLog」（https://bodylog-orcin.vercel.app）に現在実装されている全機能の一覧です。
この機能セットを **すべて維持したまま** UI を一新するためのデザイン検討に使ってください。

## 0. サービス概要と前提

- 減量（ボディメイク）のための日次トラッカー。**自然文で書くだけ・写真を撮るだけ**で、AI が食事の kcal / PFC を解析して記録するのが最大の特徴
- マルチユーザー（友人にも配布）。日本語がメイン、AI 翻訳で約100言語に切替可能
- **モバイルファースト**（iPhone ネイティブアプリは WebView でこの Web をそのまま表示。Web と UI は完全共通）
- 現行デザイン: ダークテーマ（背景 #0c131c、アクセントは teal）。数値が多いので **等幅数字（tabular-nums）** を多用
- 技術: Next.js (App Router) + Supabase。1画面 = 1ルート

## 1. 画面構成（5タブ + 共通シェル）

| タブ | ルート | 役割 |
|---|---|---|
| 入力 | /log | 毎日の記録（メイン画面・起動時に開く） |
| ダッシュボード | /dashboard | KPI・グラフ・進捗判定 |
| 目標 | /goal | 目標設定・チートデイ管理 |
| 食品 | /foods | マイ食品（よく食べる物）の登録 |
| 設定 | /settings | プロフィール・言語・通知ほか |

**共通シェル（全画面共通）**
- トップバー: ロゴ「BodyLog」/ 表示名 / 🔄更新ボタン（ページ再読込）/ ログアウト
- タブに**未入力バッジ（赤ドット）**: 体写真が1枚もない→「入力」タブ、目標未設定→「目標」タブ
- 初回ユーザー向け**ナッジポップアップ**: 「体写真を入れよう」「目標を入れよう」への導線（あとで、で消せる）
- **オフラインバナー**: 圏外時「📡 オフライン表示中—前回のデータを表示しています」
- 全ページ stale-while-revalidate: **前回データを即表示→裏で最新に差し替え**（スケルトン/スピナーはほぼ不要の設計）

## 2. 入力タブ /log（メイン画面）

### 2-1. 日付と記録入力
- 日付ピッカー（過去日の記録・修正も可能）
- **自然文テキストエリア**: 「牛丼並盛とサラダ。ジムで1時間筋トレ。体重70.2kg。気分は良い」のように自由に書く
- **写真添付 最大4枚**（食事写真・栄養成分表示の写真）。ネイティブではカメラ/ライブラリ選択
- **🤖 AIで解析ボタン**: テキスト+写真から品目リスト（名前・量・kcal・P/F/C）、体重、運動強度、気分を抽出
  - 残り回数表示「（今日あと n 回）」— 無料枠 1日15回。残り3回以下で注意書き
- 解析結果の**品目リスト編集**: 品目ごとの数量変更（0.5倍・2倍など）・削除・手修正
- **マイ食品チップ**: 登録済みの食品をワンタップで品目に追加（設定量=1回分で入る）
- **手動編集モード**: AI を使わず kcal / P / F / C / 運動レベル / 補正kcal / 体重 / 気分を直接入力
- 運動レベルは5段階: オフ(+0) / 軽い(+30) / 通常(+150) / 高(+400) / 特大(+800) kcal + 手動補正
- **保存**: 保存後に「ここまでの摂取 / 今日の目安 / 差 / 判定」を即フィードバック

### 2-2. 当日フィード（記録一覧）
- その日の記録を**時刻付きフィード**で表示（🍽 品目要約 kcal / 🏃 運動 / メモ冒頭）
- 各記録の **✎ 編集**（上書き）と **× 削除**（合計は自動再計算）
- **オフライン対応**: 圏外で保存→「⏳未同期」マーク付きで即表示→通信回復時に自動送信（「📶 n件を同期しました」）。未同期は×で取り消し可能
- 旧形式（1日1まとめ）の過去データも表示・自動移行

### 2-3. 体写真セクション（同画面下部）
- 体の写真をアップ → **AI が体脂肪率を推定**しコメント付きで返す
- 日付別の写真一覧（本人のみ閲覧可能である旨の注記あり）
- **🔍 前回と比較**: 2枚以上あると AI が変化を比較講評
- 目標体脂肪率とメモ（目標タブで設定）が判定の文脈に使われる

## 3. ダッシュボード /dashboard

### 3-1. KPI カード（5枚）
1. **目標との進捗**（最重要）: 「n日 先行 / n日 遅れ / 順調」+ 標準体重 vs 実測。色分け（先行=緑、遅れ=赤）
2. **体重**: 最新値 + 記録開始からの増減（▼▲色分き）
3. **直近7日収支**: 合計kcal + 標準ペース比
4. **累計収支**: 合計kcal + 脂肪換算 約n kg
5. **目安kcal**: 基礎代謝×生活係数（運動日は自動加算の注記）

### 3-2. 計画カード「標準進捗 vs 実績」（目標設定時のみ）
- 残り kg / 残り日数 / 必要総赤字
- **必要な赤字/日（チートデイ込み）** と通常換算
- **通常日のおすすめ摂取 kcal**
- 実現性アラート: 赤字が 700kcal/日超→⚠警告、1,000超→🚨目標見直し提案

### 3-3. グラフ
- **体重・体脂肪率の推移と計画**: 実測体重の折れ線 + 目標への計画ライン + チートデイ🍺マーカー + 体脂肪率の推移 + 目標体脂肪率ライン
- **カロリー収支の積み上げ**（累積グラフ）: 「どれだけ得したか」を可視化、今日マーカー付き

### 3-4. 実績テーブルとバー
- **直近14日テーブル**: 日付(曜日) / 運動 / 体重 / 摂取 / 目安 / 差 / 判定ピル
- **判定は5段階の色分けピル**:
  - 不足注意（−501以下）= 青
  - OK（−500〜−300）= 緑
  - ▲（−299〜−101）= teal
  - ×（−100〜+100）= 黄
  - NG（+101以上）= 赤
- **直近14日 収支バーチャート**: 中央0の左右バー、OKゾーン帯表示、凡例付き
- 記録ゼロなら「まだ記録がありません→入力タブへ」の空状態

## 4. 目標タブ /goal

- **目標設定フォーム**: 目標日 / 目標体重 / 目標体脂肪率(任意) / メモ(なりたい姿) 
- **チートデイの取り返し方**: 目標日まで均等 or 7/14/30日で取り返す（吸収日数）
- **PFC 設定**: たんぱく質 g/kg（既定2.0）・脂質 g/kg（既定0.9）・脂質の絶対上限 g/日(任意) → 入力タブ・ダッシュボードの目標PFCに反映
- **イベント（チートデイ）管理**: 日付・タイトル・追加kcal（既定800）を登録 → 計画とグラフに反映。一覧・削除
- 現在の最新体重・計画のプレビュー表示

## 5. 食品タブ /foods（マイ食品）

- 自然文 + 写真（栄養成分表示の撮影OK）→ **AI解析で品目化** → 名前を付けて登録
- 品目の手修正、品目なしの手動登録（kcal/PFC直接入力）
- **1回分の量**設定（登録全量に対する割合。例: 1/6 鍋）→ 入力タブのチップからワンタップ追加時に使われる
- 登録済み一覧（名前・単位・kcal・PFC）・削除
- AI 残り回数表示は入力タブと共通

## 6. 設定タブ /settings

- **プロフィール**: 表示名 / 性別 / 身長 / 年齢 / 生活係数（基礎代謝計算に使用）
- **言語 / Language**: 検索ボックスで約100言語から選択 → **UI全体をAI翻訳**（キャッシュされ2回目以降は即時）
- **マイ食品**への導線
- **過去データの一括取込**（JSON貼り付け）
- **🔔 通知**:
  - リマインドメール（3日間未入力で届く）の ON/OFF
  - 【ネイティブアプリのみ表示】毎日のローカル通知リマインド ON/OFF + 時刻設定
- **規約・ポリシー**へのリンク（/terms /privacy）
- **アカウント削除**: 「削除」と入力して確認 → 全データ削除

## 7. 認証・オンボーディング

- **/login**: メール+パスワード。ログイン / 新規登録の切替、利用規約・プライバシーへのリンク
- **/onboarding**: 初回のみ。ニックネーム・性別・身長・年齢・初期体重・生活係数を入力して開始

## 8. ネイティブアプリ限定の挙動（iOS / Capacitor）

※ UI は Web と共通。以下が上乗せされる
- 保存成功などで**ハプティクス**（触覚フィードバック）
- 写真ボタンで**ネイティブカメラ/フォトピッカー**
- **アプリアイコンバッジ**: 今日未記録なら「1」、記録すると消える
- **毎日のローカル通知**（設定タブでON/OFF・時刻）
- スプラッシュスクリーン、ステータスバー配色

## 9. デザイン時に考慮してほしい「状態」一覧

| 状態 | 現在の見せ方 |
|---|---|
| 初回・データなし | 空状態カード + ナッジポップアップ + タブ赤ドット |
| 読み込み | 前回キャッシュを即表示（真っ白な読み込み画面を極力出さない） |
| オフライン | 上部バナー + 保存は「⏳未同期」で即時反映 → 復帰時に自動同期トースト |
| AI残数わずか/ゼロ | ボタン脇の残数チップ、0で解析ボタン無効 |
| 保存成功/失敗 | インラインメッセージ（ok=緑 / err=赤） |
| 目標が非現実的 | ⚠/🚨 の警告カード |
| 判定 | 5色のピル（青/緑/teal/黄/赤） |

## 10. リニューアルの制約

- 機能の削除はしない（上記すべてが新UIに載ること）
- モバイルファースト必須（375px基準）。iPhoneアプリとしてもそのまま表示される
- 数値表示は等幅数字。日本語UI（AI翻訳で多言語化されるため、テキストはコンポーネント化された短い文言が望ましい）
- 5タブ構成は変更提案OK（例: 入力+ダッシュボード統合、FAB化など）。ただし「入力への最短導線」が最優先
